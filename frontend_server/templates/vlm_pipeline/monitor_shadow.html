How to review

1. Accept: Main decision is good. No popup is required.
2. Decline: Main decision is wrong. Fill popup details for error analysis.
3. NotDecide (unsure): Evidence is weak or ambiguous. Fill popup details for follow-up.
4. Edit/Override: If feedback was wrong, click Edit/Override in the row and resubmit.

Decline/NotDecide popup fields

Note: What was wrong/unclear in this event. Add factual observations first.
Error Type: Pick root cause (`false_positive`, `false_negative`, `wrong_class`, `weak_reason`, `other`).
Missed Focus: Comma-separated cues the model missed (max 12). Use the "Detected Cues" column as reference.
Suggestion: Optional prompt/policy improvement idea for next refine cycle.
Feedback

How to fill this popup

Decline/NotDecide(unsure) feedback is used to improve shadow prompts. Be concrete and factual.
Decline 입력 순서: 1) Note 2) Error Type 3) Missed Focus 4) Suggestion(선택)
Recommended minimum for decline: Note + Error Type.
Recommended minimum for not_decide: Note + Missed Focus.
Note template: [관찰 사실] + [오판 이유] + [기대 라벨]
Example: 영수증 교환만 보이고 지폐 패턴이 없어 CASH_TRANSACTION 아님
